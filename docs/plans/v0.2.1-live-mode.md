# Peek v0.2.1 ‚Äî Live Mode + Morning Fixes

**Goal:** Ship Peek such that typing `/peek_start NAME` in any Claude Code session immediately shows the session in Peek's UI, with correct live-updating timeline, bookmark, and tokens. Must work for any GitHub user via `npx peek install`.

**Architecture:** Add `peek watch` daemon (fs.watch on `~/.claude/projects/`), SSE endpoint, `POST /api/markers`, `/peek_start` + `/peek_end` slash commands, `peek install` command. Fix morning-review UI issues.

**Tech Stack:** Node, Express (existing), chokidar for file watching, EventSource for SSE, bash for slash command scripts.

## ¬ßMENTAL-MODEL

What a first-time user will try in the first 2 minutes:
1. **Open session** ‚Üí works (v0.2.0)
2. **Look at timeline** ‚Üí works (v0.2.0)
3. **Record a live session** ‚Üí NEW: `/peek_start NAME` + watch appear live
4. **Find today's session** ‚Üí NEW: `peek watch` auto-detects
5. **Understand REC** ‚Üí FIX: rename + tooltip or remove

All 5 must work before ship.

## ¬ßDATA-MAPPING (verified from real JSONL)

- `~/.claude/projects/<slug>/<session-id>.jsonl` ‚Äî one JSONL per session, appended per event
- Each line is a JSON event with `type`, `uuid`, `timestamp`, and content
- `user_prompt` events have content under `message.content[0].text`
- Marker regex: `/^\s*(?:@|\/)peek[-_](start|end)(?:\s+(.+))?\s*$/i` ‚Äî matches both `@peek-start NAME` and `/peek_start NAME`

> **L2.4-critical post-ship fix (2026-04-19)** ‚Äî the original L2.4
> implementation summed `span.tokensConsumed` per turnId as the source of
> `max_per_turn`. Spans only carry per-tool content tokens (the string
> "ls -la" ‚âà 5 tokens), so on real Claude Code sessions the gauge under-
> reported real context-window pressure by ~40x (seen: 17,920 / 200,000
> = 9 % when actual usage was 738,732 / 200,000 = 369 %). The reconciler
> had been flagging this as `Turn.usage=994,613 vs sum(spans)=6,864`
> (99.31 % drift) since v0.2 shipped.
>
> Fix: `/api/sessions/:id/events` now prepends `turn` wire events carrying
> `usage` (input + output + cacheCreation + cacheRead). The UI's
> `computeContextGaugeStats` prefers those when present (same number
> `parentReported` carries in `server/pipeline/self-check.ts` ‚Äî and what
> Claude Code's `/usage` command shows). Falls back to span-sum for legacy
> sessions imported before turn events were on the wire. See
> `tests/unit/context-gauge-compute.test.ts` "REGRESSION: reads turn.usage
> when available" for the pinned contract.
